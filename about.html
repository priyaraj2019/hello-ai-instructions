    What is AI Instructions Lab?

    AI Instructions Lab is a beginner-friendly practice repository that teaches you how to guide GitHub Copilot using repository instructions and AI skills. This is a simple, real-world example you can learn from and adapt to your own projects.

    Back to home

    What you learn here

    This repository demonstrates three core concepts that help you work better with GitHub Copilot:

    • 📋 Repository instructions

      How to write clear, always-on guidance that shapes how Copilot behaves across your entire project. See the .github/copilot-instructions.md file.

    • 🧠 AI skills (playbooks)

      How to create task-specific skills that help Copilot understand when and how to apply best practices. Skills are triggered when relevant.

    • ✨ Practical examples

      Real examples like improving a landing page, adding learning notes, and building a brand system—all shown on this very site.

    How this repo teaches you

    Step 1: Repository instructions guide Copilot

    The .github/copilot-instructions.md file contains always-on guidance. Every time Copilot interacts with this repository, it reads these instructions. They set the tone: keep things simple, use plain HTML and CSS, stay beginner-friendly.

    Step 2: Skills are task-specific playbooks

    Skills live in .github/skills/ and provide detailed guidance for specific tasks. For example:

    • landing-page-review skill improves clarity and accessibility
    • learning-note skill adds educational content
    • brand-system skill ensures visual consistency

    Step 3: See the results on this page

    Every section of this site was built using Copilot with repository instructions and skills. You can see exactly what was improved and why. This helps you understand how to use these tools in your own projects.

    Why this matters for beginners

    If you're new to GitHub Copilot, you might think: "Copilot is just an autocomplete tool." But Copilot is smarter than that. It reads your project's context and guidelines to give you better advice.

    Repository instructions tell Copilot what matters to you: "Keep it simple. No frameworks. Be beginner-friendly."

    Skills teach Copilot how to solve specific problems: "When improving a landing page, think about clarity, accessibility, and visual layout."

    Together, they create a personalized AI assistant that understands your project and your values.

    Next steps

    1. Read the repo instructions: Open .github/copilot-instructions.md to see how Copilot is being guided.
    2. Explore the skills: Look in .github/skills/ to see different task-specific playbooks.
    3. Try it yourself: Ask Copilot to improve a section of this page or add a new feature. Notice how it respects the repository instructions.
    4. Adapt to your projects: Create your own .github/copilot-instructions.md file and skills in your repositories.
